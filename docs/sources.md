# Data sources

All sources are public. Commercial SCA platforms are not used.

The live (`source_mode=live`) branch of the pipeline fans out to the
endpoints below; the fixture (`source_mode=fixture`) branch reads
committed CSVs under `pipeline/fixtures/` and hits nothing external.

## Source-health plumbing

Every live raw asset records one row to `raw.source_health` with `status`
(`ok` / `degraded` / `failed`), `row_count`, `latency_ms`, and the first
error message (if any). `mart.source_health` rolls those into the
per-weekly-snapshot coverage table consumed by the showcase.

Failure → confidence mapping is codified once in
[`pipeline/lib/sources.py`](../pipeline/lib/sources.py); downstream marts
never re-derive it.

## Usage rules

1. **Cache raw extracts** per reporting window on disk (`.cache/http/`) and in BigQuery/DuckDB staging tables. Do not re-hit public APIs on rerun.
2. **Source failure → reduced confidence or source-health warning.** Never convert a failure into negative package evidence.
3. **Every public methodology/coverage surface names the source category** behind each evidence signal.
4. **GitHub auth** is required for the live MVP (60 req/hr unauthenticated cannot cover 500 packages × several endpoints); optional for small dev runs. Prefer `INGEST_TOKEN`; `GITHUB_INGEST_TOKEN` remains supported for compatibility. Unauthenticated runs fall back gracefully and mark affected sources `degraded`.
5. **Commercial SCA is forbidden as source data.** Use OpenSSF Scorecard as the public, complementary signal instead.
6. **BigQuery is cold-path only.** The weekly hot path must not issue BigQuery jobs; only `scripts/refresh_universe.py` may do so, and only on operator demand.
